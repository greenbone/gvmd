## Glossary*

| Term                             | Definition                                                                                                                                                                                                                                             |
|----------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Target**                       | A set of hosts or IP addresses to be scanned. Includes configuration options like excluded hosts, alive test methods, and reverse DNS resolution settings.                                                                                             |
| **Asset**                        | A deduplicated, long-lived representation of a host (or logical host) across scans. Aggregates identifiers, attributes, and findings over time, and tracks fields like `last_seen`, managed state, and per-severity counters.                          |
| **Task**                         | Represents a scan configuration linked to a specific target and schedule. Tasks can be scheduled or executed immediately.                                                                                                                              |
| **Scan**                         | A full vulnerability assessment process triggered via a task, analyzing the target with specific preferences and vulnerability tests (VTs).                                                                                                            |
| **Result**                       | The outcome of a scan execution, detailing discovered vulnerabilities, associated hosts, ports, and messages.                                                                                                                                          |
| **Host Detail**                  | The outcome of a scan execution, detailing discovered vulnerabilities, associated hosts, ports, and messages.                                                                                                                                          |
| **Host**                         | An individual IP or hostname under a target. Hosts may be associated with scan results.                                                                                                                                                                |
| **Port**                         | Represents protocol-specific port ranges to be scanned (e.g., TCP/UDP). May be part of alive tests or credential-specific scans.                                                                                                                       |
| **Credential**                   | Authentication data used for authenticated scans. Supports multiple types such as username/password, SNMP, TLS certificates, or Kerberos.                                                                                                              |
| **Scan Config**                  | A template of selected vulnerability tests and preferences. Configures what VTs are run during a scan.                                                                                                                                                 |
| **Preference**                   | A template of selected vulnerability tests and preferences. Configures what VTs are run during a scan.                                                                                                                                                 |
| **Scanner**                      | The OpenVAS-compatible engine that performs the scan. May include configuration for MTLS authentication.                                                                                                                                               |
| **Scanner Type**                 | The OpenVAS-compatible engine that performs the scan. May include configuration for MTLS authentication.                                                                                                                                               |
| **Alive Test**                   | A technique used to detect if a host is responsive before scanning. Methods include ICMP ping, TCP connect, or service-based checks.                                                                                                                   |
| **Report**                       | A persistent artifact generated after scan execution, including summaries and full vulnerability details.                                                                                                                                              |
| **Compliance Report**            | A persistent artifact generated after scan execution, including summaries and full vulnerability details.                                                                                                                                              |
| **Schedule**                     | Defines when a task should run (e.g., cron-style). Can be used to trigger recurring scans automatically.                                                                                                                                               |
| **Audit**                        | A record of user-initiated actions within the system (e.g., create, update, delete). Includes metadata like user ID, action type, timestamp, IP address, and affected resources. Useful for compliance, traceability, and security auditing.           |
| **Owner**                        | Each resource (e.g., target, task) includes an `owner` field indicating the user who created it. Helps enforce access control and visibility.                                                                                                          |
| **NVT**                          | Vulnerability Tests (VTs) are individual security checks executed by the scanner. Each VT detects a specific vulnerability, misconfiguration, or system property.                                                                                      |
| **VT**                           | Vulnerability Tests (VTs) are individual security checks executed by the scanner. Each VT detects a specific vulnerability, misconfiguration, or system property.                                                                                      |
| **NVT Family**                   | Vulnerability Tests (VTs) are individual security checks executed by the scanner. Each VT detects a specific vulnerability, misconfiguration, or system property.                                                                                      |
| **VT Metadata**                  | Metadata associated with a VT, including OID (unique identifier), name, severity, QoD, family, and references (e.g., CVE, CPE, vendor advisories). This metadata ensures scans and reports provide consistent and traceable vulnerability information. |
| **QoD**                          | Quality of Detection is a metric (0–100%) representing the reliability of a VT’s detection result. Higher QoD means higher confidence that a finding reflects a real vulnerability. Used for filtering and prioritizing scan results.                  |
| **CVSS**                         | Common Vulnerability Scoring System. A standardized scoring system (0.0–10.0) that measures the severity of vulnerabilities based on impact and exploitability. Used in scan results and reports for prioritization.                                   |
| **EPSS**                         | Common Vulnerability Scoring System. A standardized scoring system (0.0–10.0) that measures the severity of vulnerabilities based on impact and exploitability. Used in scan results and reports for prioritization.                                   |
| **Severity**                     | A normalized categorization of vulnerability criticality (e.g., Low, Medium, High, Critical). Often derived from CVSS scores and used in dashboards, reports, and filtering.                                                                           |
| **Threat**                       | A normalized categorization of vulnerability criticality (e.g., Low, Medium, High, Critical). Often derived from CVSS scores and used in dashboards, reports, and filtering.                                                                           |
| **CVE**                          | A normalized categorization of vulnerability criticality (e.g., Low, Medium, High, Critical). Often derived from CVSS scores and used in dashboards, reports, and filtering.                                                                           |
| **CPE**                          | A normalized categorization of vulnerability criticality (e.g., Low, Medium, High, Critical). Often derived from CVSS scores and used in dashboards, reports, and filtering.                                                                           |
| **SCAP Data**                    | A normalized categorization of vulnerability criticality (e.g., Low, Medium, High, Critical). Often derived from CVSS scores and used in dashboards, reports, and filtering.                                                                           |
| **CERT Advisory**                | A normalized categorization of vulnerability criticality (e.g., Low, Medium, High, Critical). Often derived from CVSS scores and used in dashboards, reports, and filtering.                                                                           |
| **Closed CVE**                   | A normalized categorization of vulnerability criticality (e.g., Low, Medium, High, Critical). Often derived from CVSS scores and used in dashboards, reports, and filtering.                                                                           |
